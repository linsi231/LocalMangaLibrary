## LocalMangaLibrary v0.3.0-alpha / Alpha 3

本次更新是一次结构性重构，重点不是新增复杂功能，而是重新明确项目定位并修复早期版本的状态污染问题。

LocalMangaLibrary 从 Alpha 3 开始定位为：

> 单文件、一次性、本次运行内使用的本地漫画快速浏览工具。

---

## 主要变化

### 1. 重新明确产品定位

* 不再按长期媒体库管理器设计；
* 不再保留上次漫画库路径；
* 不再保留上次索引；
* 不再保留历史记录；
* 不再保留主题和设置参数；
* 关闭程序后清理本次运行产生的数据。

### 2. 修复本地媒体库刷新问题

修复以下问题：

* 切换漫画库后列表仍显示旧库内容；
* 当前库路径已经变化，但媒体库没有刷新；
* 合计作品数、图片数、文件数累计了之前使用过的目录；
* 刷新索引后统计数据不是当前库数据。

现在本地媒体库的合计数据只来自当前选择的漫画库。

### 3. 重构运行数据生命周期

Alpha 3 开始，运行数据只在本次 session 内有效。

关闭程序后应清理：

* 当前库路径；
* 当前索引；
* 当前统计；
* 搜索、排序、筛选状态；
* 历史阅读；
* 历史访问目录；
* 缩略图缓存；
* 阅读缓存；
* 本次运行生成的临时 JSON。

### 4. UI 页面结构重构

计划将主界面拆分为更清晰的页面结构：

* `MainWindow`：应用外壳和页面切换；
* `LibraryView`：当前媒体库；
* `HistoryView`：本次运行内历史；
* `SettingsView`：本次运行内设置、缓存状态、版本信息。

目标是避免“标题是设置，但页面仍显示媒体库”的问题。

### 5. 缓存策略调整

* 缓存只服务当前运行 session；
* 启动时创建新的 `.cache/session_*`；
* 关闭时清理本次 session；
* 不再跨运行复用旧缩略图；
* 不再保留长期缓存池；
* 清理缓存不会影响用户漫画源文件。

---

## 安全边界

本工具不会：

* 删除漫画源文件；
* 移动漫画源文件；
* 重命名漫画源文件；
* 覆盖漫画源文件；
* 向漫画源目录写入缓存或索引；
* 上传任何本地文件。

所有清理操作仅限程序自己创建的 session/cache/runtime 数据。

---

## 测试重点

建议重点测试：

1. 启动后当前库为空；
2. 启动后不显示上次漫画库路径；
3. 选择漫画库 A 后，合计只显示 A 的数据；
4. 切换漫画库 B 后，A 的列表和合计立即清空；
5. B 扫描完成后，合计只显示 B 的数据；
6. 设置页面不再显示媒体库列表；
7. 历史页面不再显示媒体库列表；
8. 关闭程序后 `.cache/session_*` 被清理；
9. 重新打开后没有上次路径、历史、设置和索引；
10. 清理缓存和关闭程序不会影响漫画源文件。

---

## 注意事项

当前仍为 alpha 版本。
本次版本主要用于验证新的产品定位、页面结构和运行数据生命周期。
如果需要长期媒体库管理、跨运行历史记录或长期索引缓存，请继续使用其它成熟漫画管理器。
